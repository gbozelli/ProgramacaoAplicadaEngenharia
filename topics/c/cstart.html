Estrutura de um programa
Um programa em C, comumente, terá a seguinte estrutura:
//declarações globais (bibliotecas)

int main(){//início do programa
  //instruções
} //fim do programa
As bibliotecas são conjuntos de códigos prontos que facilitam (e muito) a vida do programador. Basicamente, vamos importar códigos para realizar determinadas tarefas, e essas tarefas vão auxiliar a nossa escrita do programa.
A biblioteca básica de C é a stdio.h (do inglês "Standard Input/Output"). Importamos essa biblioteca para usar as funções printf() e scanf(), que são configurações de "entrada" e "saída" (lembre da entrada e saída vista no tópico anterior). Não se preocupe com seu uso e implementação agora, ambas serão explicadas futuramente.
O trecho int main() inicialmente pode parecer bastante confuso. Ele declara uma função (a função executa um conjunto de instruções) int main, que é definida dentro da linguagem, e deve ser utilizada para que o código seja executado. Se essa função não for colocada, o compilador não lê o código, e não há execução. Observe que as chaves "{}" são o que determinam o início e o fim do programa
As instruções são o 'algoritmo', o que o nosso programa deve fazer. Posteriormente, veremos que as instruções podem ser colocadas fora do escopo da função principal utilizando 'funções'. Por ora, elas serão colocadas aqui dentro, e incluem o uso de palavras-chave (os comandos da linguagem).
Vamos entender a funcionalidade do seguinte código:
#include <stdio.h>

int main(){
  printf("Estou estudando C!");
  return 0;
}
O trecho #include <stdio.h> importa a biblioteca padrão de C. Observe que há o uso do # como comando para demonstrar ao compilador que essa é uma biblioteca. Caso o # não seja colocado, o compilador provavelmente vai reportar o erro ao programador.
Vamos entender melhor o printf(). Sendo uma das funcionalidades da biblioteca stdio.h, ela imprime uma mensagem de texto na tela do computador, é a saída do nosso programa. Parece pouco funcional imprimir uma mensagem que nós já estamos escrevendo no programa. O uso de parênteses entre a mensagem serve para demarcar a mensagem que será escrita, e o ponto e vírgula indica uma quebra de linha, ou seja, que o comando dessa linha já acabaou e o compilador deve seguir para a próxima.
Por último, a linha return 0;. Essa linha é uma espécie de convenção adotada pelo compilador para compreender se o programa está rodando de forma correta. Se ele retornar 0, significa que o programa rodou de forma desejada, e se retornar outro número, há algum erro no nosso código. Sempre coloque esse trecho por último na função int main() .
Usando a função "scanf()"
Até o momento, vimos que a mensagem deve ser escrita diretamente no código, mas e se eu quiser assumir uma estrutura mais liberal e "dar" uma mensagem ao programa?. Para isso, existe a função scanf(), que recebe dados digitados no teclado. A configuração scanf() é a entrada do nosso programa. Vamos analisar o seguinte código:
#include <stdio.h>

int main(){
  char nome[100];
  printf("Digite seu nome: ");
  scanf("%s", nome);
  printf("Nome: %s", nome);
  return 0;
}
Observe que, para utilizar a função scanf() é preciso acrescentar mais coisas ao código, e muito provavelmente você tenha ficado confuso com alguma linha. Por conta disso o uso do scanf() será estudado e analisado posteriormente, mas é bom que já se saiba da existência dele.
