Estrutura de um programa
Um programa em C, comumente, terá a seguinte estrutura:
//conjunto de instruções

int main(){
  //declarações
}
Entende-se por conjunto de instruções como pedaços de códigos que fazem coisas que o C, sozinho, não faz. Basicamente, vamos importar códigos para realizar determinadas tarefas, e essas tarefas vão facilitar nossa escrita do código.
A biblioteca básica de C é a stdio.h (do inglês "Standard Input/Output"). Importamos essa biblioteca para usar as funções printf() e scanf(), que são configurações de "entrada" e "saída" (lembre da entrada e saída vista no tópico anterior). Não se preocupe com seu uso e implementação agora, ambas serão explicadas futuramente.
O trecho int main() inicialmente pode parecer bastante confuso. Ele declara uma função (a função executa um conjunto de instruções) int main, que vem de "principal". Essa função é definida dentro da linguagem, e deve ser utilizada para que o código seja executado. Se essa função não for colocada, o compilador não lê o código, e não há execução.
E as declarações? São as instruções da receita de bolo. Posteriormente, veremos que declarações podem ser colocadas fora do escopo da função principal. Por ora, elas serão colocadas aqui dentro, e incluem declarações de variáveis (lembre de variáveis em matemática), estruturas de repetição (loops), condicionais, operadores lógicos, entre outras.
Vamos entender a funcionalidade do seguinte código:
#include <stdio.h>

int main(){
  printf("Estou estudando C!");
  return 0;
}
O trecho #include <stdio.h> importa a biblioteca padrão, isso já foi mencionado. Observe que há o uso do # como "comando" para demonstrar ao compilador que esse é um conjunto a ser importado. Caso o # não seja colocado, o compilador não reconhece o resto do código include <stdio.h>(e provavelmente vai reportar o erro ao programador).
Vamos entender melhor o printf(). Sendo uma das funcionalidades da biblioteca stdio.h, ela é a saída do nosso programa, nosso resultado, o que nos é retornado. Parece pouco funcional imprimir uma mensagem que nós já estamos escrevendo no programa, mas tente pensar no seguinte: ofereço ao programa uma entrada (mensagem), e ele retorna uma saída (a mesma mensagem).
Por último, a linha return 0;. Essa linha é uma espécie de convenção adotada pelo compilador para compreender se o programa está rodando de forma correta. Se ele retornar 0, significa que o programa rodou de forma desejada, e se retornar outro número, há algum erro no nosso código. Sempre coloque esse trecho por último na função int main() .
E agora?
Até o momento, apenas vimos a estrutura do código de uma forma mais livre. Você deve estar pensando "Mas o que significam as {}chaves, ;ponto e vírgulas? o que extamente return faz?". Esses pensamentos de início são muito comuns, porque não estamos familiarizados com o "pensar" de um computador, com a sua sintaxe rígida, pois pensamos em formas de palavras, imagens, sons, e não em comandos. Eventualmente, com a escrita e prática de códigos, isso será natural.
